# Safety and Scope Boundaries

## Purpose

This file defines the safety and scope boundaries the website should respect.

The website should support Asmar Partners’ positioning as a safe AI adoption and workflow improvement firm. It should not imply legal, tax, compliance certification, incident response, security audit, or production hosting services unless separately scoped.

## Prototype and Advisory Boundary

The website workflow suggestion tool is a preview, not a full assessment.

Use language like:

```text
This preview is based on limited information and is not a complete workflow assessment.
```

```text
Asmar Partners can help evaluate the actual process, data sensitivity, approved tools, governance needs, and pilot plan.
```

Do not imply:

- Guaranteed ROI
- Guaranteed accuracy
- Complete legal/compliance coverage
- Production readiness from a website form
- Automated decisions without review

## Human Review Principle

AI-generated outputs should be reviewed by a qualified human before being used for:

- Client-facing communication
- Business decisions
- Compliance decisions
- Legal or tax matters
- Financial recommendations
- Operational commitments
- System changes
- Access control changes

The workflow suggestion tool should always include human-review points.

## Sensitive Data Restrictions

Do not ask visitors to enter or upload sensitive data.

The form should warn users not to include:

- Client names
- Financial records
- Bank statements
- Payroll records
- Tax records
- Social Security numbers
- Tax IDs
- Credentials
- API keys
- Access tokens
- Private keys
- Health information
- Legal privileged information
- Confidential business records
- Regulated data

## Public AI Tool Warning

Do not recommend uploading sensitive client financial records, tax records, payroll data, bank statements, credentials, or regulated information into public AI tools.

If a workflow involves sensitive information, recommend:

- Sanitized examples for testing
- Approved business workspace
- Client-owned platform
- Access controls
- Human review
- Data retention rules
- Legal/compliance review where appropriate

## Out-of-Scope Website Claims

The website should not present Asmar Partners as providing these services by default:

- Legal advice
- Tax advice
- Compliance certification
- Formal risk attestation
- Security audits
- Penetration testing
- Incident response
- Breach investigation
- 24/7 production support
- Managed IT services
- Full custom application development
- Private RAG implementation
- Vector database implementation
- CRM/accounting/email integrations

These may require separate scope, written agreement, and appropriate professional review.

## Safe Recommendation Pattern

When recommending a workflow, prefer this structure:

1. What can be assisted safely
2. What should remain human-reviewed
3. What should not be automated yet
4. What data should not be used
5. What a controlled pilot would test
6. What success metrics should be tracked

## Example Safety Language

```text
Use general examples only. Do not enter client names, financial records, payroll data, tax records, credentials, or confidential information.
```

```text
This preview is not legal, tax, compliance, security audit, or incident response advice.
```

```text
AI-assisted workflows should be reviewed, tested, and approved before operational use.
```

```text
Asmar Partners helps firms define practical guardrails before adopting AI in real business processes.
```
