# ROI and Measurement Principles

## Purpose

The website should communicate that Asmar Partners focuses on measurable workflow improvement, not vague AI adoption.

The workflow suggestion tool should recommend success metrics for each suggested workflow.

## Measurement Chain

Use this value chain:

```text
Adoption → Usage → Time Saved → Cost → Risk Controls → ROI
```

The goal is not just to show that employees are using AI. The goal is to show whether the workflow creates measurable business value while staying within safe usage boundaries.

## What to Measure

For each recommended workflow, suggest practical metrics such as:

- Number of workflow runs
- Time saved per run
- Manual time before AI assistance
- AI-assisted time after workflow change
- Human review time
- Staff edit rate
- Error/rework rate
- Escalation rate
- User adoption
- User satisfaction
- Client response time, when relevant
- Number of follow-ups reduced
- Cost per workflow run, where applicable

## ROI Caution

Do not claim exact ROI from the website preview.

Use language like:

```text
ROI figures should be estimated from agreed workflow-volume, task-time, and labor-rate assumptions, then validated during a pilot.
```

```text
The preview suggests what could be measured; a real engagement would validate baseline time, usage, cost, and review requirements.
```

## Suggested Website Language

```text
We do not just build the workflow. We help define how it should be measured.
```

```text
A controlled pilot should track usage, time saved, review effort, risk controls, and adoption.
```

```text
The safest first workflow is usually one that is repetitive, measurable, and easy to review before wider rollout.
```

## Suggested Metrics by Workflow Type

### Internal Status Summary Workflow

- Time spent creating weekly summaries
- Number of manual follow-ups reduced
- Manager review time
- Missed blockers identified
- Staff edit rate

### Client Document Collection Workflow

- Number of missing-document follow-ups
- Average time to prepare document request
- Client response time
- Staff review time
- Rework rate

### SOP Drafting Workflow

- Number of SOP drafts created
- Time to create first draft
- Manager edit rate
- Staff questions reduced
- SOP adoption rate

### Meeting Notes to Action Plan Workflow

- Time spent summarizing notes
- Number of action items captured
- Missed follow-ups reduced
- Review/edit rate
- User satisfaction

### AI Usage Review Workflow

- Number of AI use cases submitted
- Number approved, rejected, or escalated
- Sensitive-data risks identified
- Time to review use cases
- Training opportunities identified

## Dashboard Direction

Do not build a dashboard in the first website version.

The website should only suggest measurement ideas. A full AI usage and ROI dashboard can be a future service or add-on.
