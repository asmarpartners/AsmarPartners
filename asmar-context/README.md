# Asmar Partners Website Context

These files are context for building the Asmar Partners website in the GitHub repository:

```text
https://github.com/asmarpartners/AsmarPartners
```

Use these files to understand positioning, copy, safety boundaries, and the workflow suggestion tool.

## Build Intent

Build a small, professional landing page with one interactive internal workflow suggestion tool.

This should not become a large marketing website, chatbot demo, dashboard product, login portal, or complex application.

## Core Direction

The website should communicate that Asmar Partners helps firms:

- Identify repetitive internal business workflows
- Apply AI and automation safely
- Define governance guardrails
- Require human review where appropriate
- Launch measurable workflow pilots
- Avoid unnecessary security, data, and operational risk

## Important Constraint

The workflow suggestion tool should feel like an internal business workflow assessment, not a chatbot.

Use language like:

- Internal workflow
- Workflow suggestion plan
- Safe AI adoption
- Business process improvement
- Governance guardrails
- Human review
- Measurable pilot
- Automation readiness

Avoid language like:

- Chatbot
- Bot
- Ask anything
- Talk to AI
- Fully automated decisions
- AI magic

## Recommended Site Scope

Build one main landing page with:

1. Header
2. Hero
3. Short services/process section
4. Workflow Suggestion Preview tool
5. Final call-to-action
6. Footer

Do not copy long internal planning, legal, or strategy language directly onto the site. Use these files to guide concise, client-facing copy.
